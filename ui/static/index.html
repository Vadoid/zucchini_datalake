BQ rows is the current row count (distinct primary key), so re-syncing a table won't inflate it. Lag is how old each table's newest row is, which reads high for tables nobody writes to since their last change was the backfill.
Turning a table on adds it to the publication and the stream. Datastream backfills it and creates the matching alloydb_iceberg.public_<table> table. New tables show up on their own (the page polls every 3s), or click Refresh.
